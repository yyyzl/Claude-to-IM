# 前端质量规范

> 当前仓库暂无前端代码。本文件用于防止未来 UI 层侵蚀桥接核心边界。

---

## 当前状态

目前没有前端测试、组件测试或页面质量基线。现有质量基线主要针对桥接核心：

- TypeScript 严格类型
- `node:test`
- 宿主契约与边界校验

---

## 如果未来新增前端

新增 UI 时至少应满足：

- 与桥接核心清晰分层
- 正式使用宿主暴露的数据与操作接口
- 为关键页面或组件建立可重复验证的测试方式

---

## 必须遵守

- UI 只做展示、交互和宿主控制
- 文档、类型和测试要与真实实现保持同步
- 涉及桥接状态展示时，要以正式接口为唯一数据入口

---

## 禁止事项

- 前端直接依赖桥接内部实现细节
- 前端自己维护另一套桥接状态真相源
- 把规范写成理想化口号，却与真实代码结构不一致

---
